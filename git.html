Git e GitHub

Controle de versão e colaboração em projetos de desenvolvimento

O que é Git?

Git é um sistema que ajuda a controlar as versões dos arquivos. Ele permite que várias pessoas trabalhem juntas em projetos de software, mantendo o histórico de todas as alterações realizadas.

O que é o GitHub

GitHub é uma plataforma online que armazena projetos que usam Git. Ela facilita a colaboração e o armazenamento de código, permitindo que desenvolvedores trabalhem juntos em projetos de qualquer lugar do mundo.

Diferenças entre Git e GitHub

Git: É uma ferramenta que você usa no seu computador para gerenciar versões.
GitHub: É um site que armazena repositórios Git e ajuda na colaboração.

Comandos Git Essenciais

git init – começa um novo repositório
git status – verifica o estado do repositório
git add – adiciona arquivos para serem salvos
git commit -m "mensagem" – salva as mudanças feitas
git push – envia as alterações para o GitHub
git pull – baixa as atualizações do repositório remoto
git clone [url] – baixa um repositório existente

Como criar um repositório no GitHub

Para criar um repositório, vá para o GitHub, clique em New Repository, preencha o nome e a descrição, e clique em Create Repository.

Como conectar um projeto existente da máquina ao GitHub

1
Abra o projeto no terminal
2
Use git init para iniciar o repositório
3
Conecte ao GitHub com git remote add origin <link_do_repositório>
4
Adicione arquivos e faça o primeiro commit

Como criar uma versão com Git

Use os comandos:

git add . – adiciona todos os arquivos para serem salvos
git commit -m "mensagem" – cria uma nova versão do projeto

Como enviar da máquina para o GitHub

Após fazer o commit, use:
git push origin main
(ou o nome da sua branch).

Fluxo de Trabalho Básico

1
Inicializar repositório
git init
2
Adicionar arquivos
git add .
3
Fazer commit
git commit -m "mensagem"
4
Conectar ao GitHub
git remote add origin [url]
5
Enviar para o GitHub
git push -u origin main

Dicas Importantes

  • Commite frequentemente com mensagens claras e descritivas
  • Use branches para desenvolver novas funcionalidades
  • Sempre faça git pull antes do git push
  • Resolva conflitos de merge com atenção
  • Use .gitignore para arquivos desnecessários
  • Mantenha seu repositório organizado e com histórico limpo

Comandos Úteis para Troubleshooting

git log --oneline
# Mostra histórico compacto
git diff
# Mostra diferenças não commitadas
git reset --hard HEAD
# Desfaz todas as mudanças não commitadas
git branch
# Lista todas as branches
